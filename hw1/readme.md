# Car Price Predictor

## 1. Что было сделано
- проведена обработка датасета, дедубликация, очистка данных, заполнение пропусков
- проведен разведочный анализ данных
- Созданы дополнительные признаки (отношения, квадраты, возраст машины и тд.).
- Построен пайплайн OHE + масштабирование и обучены разные варианты моделей
- Создан сервис на Streamlit (предикт по вводу параметров вручную, через загрузку CSV, визуализация данных и важности признаков)

## 2. Полученные результаты
- из данных выжат R2 ~0.96 на линейной регрессии
- реализован сервис на Streamlit

## 3. Что повлияло сильнее всего
1. Сгенерированы новые признаки по существующим:
- осмысленное добавление произведений и частных признаков (r2 увеличилась ~0.11)
- добавление квадратов отдельных признаков (r2 увеличилась на несколько пп)
2. Добавлены новые признаки:
- добавлена информация о марке, моделе и спецификациях автомобиля  (r2 увеличилась более, чем на 10 пп) - в 2 разы быстрее OHE по имени авто
- добавлена информация о полном приводе - незначительный прирост r2
- генерация новых признаков по порогам метрику не улучшила
- добавление возраста, квадрата возраста машины и произведение возраста на можность улучшило r2
3. Доработка существующих признаков
- по другому удалены дубли (оставлены авто с одинаковыми параметрами и разными ценами) - незначительный прирост
- клипование верхнего и нижнего 1% можности несколько снизили переобучение
- удаление авторского варианта заполнения пропусков мощности несколько улучшило метрику на тесте
- заполнение пропусков медианой по бренду, затем глобальной медианой несколько улучшило метрику
- добавление флагов пропусков незначительно улучшило метрику
- логарифмирование признаков не улучшило метрику
- цены на некоторые авто кажутся выбросами, но удаление этих авто роняет метрику

## 4. Что сделать не удалось
- регрессия получилась очень раздутая по фичам, не получались сделать на малом количестве

## 5. Оценка сервиса
- на Streamlit удобно быстро сделать прототип для демонстрации заказчику
- интерфейс простой и удобный
- некоторые графики визуально перегружают
- возможно стоит заполнять пропуски по другому
